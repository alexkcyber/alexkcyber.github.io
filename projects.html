Virtual Strategic Missile Integration Complex

There is a need to be able to perform cyber assessments on the Air Force's Strategic Missile Integration Complex (SMIC). This is difficult to accomplish due to the fact that the SMIC was created before the invention of internet.

A script was developed that automates the validation/verification of simulated weapon system circuits. The script automatically performs comprehensive tests and returns the results in a matter of seconds. This was an important effort because manual testing took many hours of tedious work and the script reduced this time greatly.
Colorado Cyber Innovation Center

Architected a new solution to enhance the current network infrastructure. This aimed for multiple virtual local area networks to enable the virtual separation of physical labs.

This was important because the new architecture would allow for simultaneous cyber assessments across all lab locations. This is due to the fact that cyber assessments can be very heavy on a network, so by separating locations on separate VLANs, this gives the opportunity for multiple cyber assessments without the worry of network congestion.
Cooperative Research and Development Agreement

The power usage and thermal output of the equipment in the cyber lab was researched. This enabled the creation of detailed documentation that outlined the typical and maximum power usage/thermal output.

As a result, this ensured that the cooling and power was sufficient to handle the equipment. This is important because servers can cost $30,000, and if they were to overheat and become permanetly damaged, this could be a large setback.
Networking Error Control

Developed a network error control program written in the C programming language. The program would input a bit stream of a message and then convert it to a frame.

It would then use Cyclic Redundancy Check (CRC) as well as Hamming for single bit error detection and correction.

Vulnerability Documentation

Performed enumeration on multiple vulnerable machines. Would find various exploits and security flaws in order to gain unauthorized access.

Created detailed documentation on the various attacks used and also how to prevent them from happening again in the future. The documentation followed a five stage sequence of an attack:
  1. Reconnaissance
  2. Initial Exploitation
  3. Establish Persistence

Socket Programming

Wrote a program in the C programming language that used socket programming to transfer files between a client and a server. The Transmission Control Protocol (TCP) was used.

The server would listen for a client to connect and for the client to request a file. Then, the server would send the data of that file to the client, where the client would put the bytes together to create the original and unmodified version of the same file.
